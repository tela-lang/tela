# Contributing to Tela

Thank you for your interest in contributing! Here's how to get started.

## Setup

```bash
git clone https://github.com/yourname/tela
cd tela
npm install
```

## Project Structure

```
tela/
  bin/tela.js          CLI entrypoint
  src/
    tokenizer.js       Lexer — turns .tela source into tokens
    parser.js          Parser — tokens → AST
    ast-types.js       AST node type constants
    compiler.js        Compiler — AST → JS + CSS
    runtime.js         Runtime — reactive DOM engine
    index.js           Node.js API entrypoint
    test/              All tests
  dist/
    runtime.umd.js     Built UMD bundle (generated by npm run build)
  templates/
    init/              Files scaffolded by `tela init`
  examples/
    petclinic-tela/    Spring Boot + Tela demo app
  docs/                Documentation site (built with Tela!)
```

## Running Tests

```bash
npm test
```

## Building the Runtime

```bash
npm run build
```

This generates `dist/runtime.umd.js` from `src/runtime.js`.

## Submitting Changes

1. Fork the repo
2. Create a branch: `git checkout -b feat/my-feature`
3. Make your changes and add tests
4. Run `npm test` — all tests must pass
5. Open a pull request

## Reporting Bugs

Open an issue with:
- The `.tela` source that causes the problem
- Expected vs actual output
- Node.js version (`node --version`)
